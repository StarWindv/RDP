# P1.5 完整管道和重定向系统实现计划

## 当前状态分析

### 已存在的功能
1. **Parser**：能够解析管道符号 `|` 和基本的重定向符号
2. **SSA IR**：有基本的管道和重定向指令定义
3. **SSA IR生成器**：有基本的管道生成逻辑
4. **SSA执行器**：有基本的管道执行框架

### 缺失的功能
1. **完整的重定向类型支持**：
   - 输出重定向 `>`, `>>`
   - 输入重定向 `<`
   - 错误重定向 `2>`, `2>>`
   - 文件描述符操作 `n>&m`, `n<&m`, `>&-`, `<&-`
   - Here文档 `<< EOF`
   - 读写重定向 `<>`

2. **管道执行逻辑**：
   - 多命令管道执行 `cmd1 | cmd2 | cmd3`
   - 正确的进程创建和连接
   - 文件描述符管理
   - 管道错误处理

3. **平台支持**：
   - Unix：使用nix和os_pipe库
   - Windows：使用os_pipe库（跨平台支持）

## 实现步骤

### 第4步：编写测试

## 具体实现细节

### 1. Parser改进
- 在 `parse_simple_command()` 中处理重定向符号
- 支持 `n>`, `n>>`, `n<`, `n>&m` 等语法
- 解析Here文档内容

### 2. SSA IR生成器改进
- `generate_pipeline()`：正确处理多命令管道
- `generate_redirection()`：生成所有重定向类型的指令
- 处理文件描述符数字

### 3. SSA执行器改进
- `execute_instruction()`：实现 `CreatePipe`, `DupFd`, `CloseFd`, `Redirect` 指令
- 实现进程创建和管道连接
- 实现文件打开和重定向

### 4. 平台抽象
- 使用 `os_pipe` 库进行跨平台管道创建
- 使用 `nix` 库处理Unix特定的文件描述符操作
- 为Windows提供替代实现

## 验收标准

完成P1.5后，应能通过以下测试：

1. ✅ `echo hello | grep hello` - 简单管道

8. ✅ `echo test 2>&1` - 文件描述符复制
