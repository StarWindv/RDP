## 🔑 关键决策

### 1. 优先级调整 (相比初始计划)

- **原始**: Phase 1 → Phase 2 → Phase 3 → Phase 4
- **调整**: Phase 1 → Phase 3 → Phase 4 → Phase 2
- **原因**: 脚本执行可用后立即做POSIX兼容测试，优化Phase 4后再加交互功能

### 2. 技术栈约束

```
禁用:    libc::* 直接系统调用
优先:    - os_pipe (管道)
        - nix crate (POSIX接口)
        - glob crate (路径名匹配)
        - users crate (用户信息)
        - 其他Rust生态库
```

### 3. 开发模式

- **提交粒度**: 小功能级别
- **提交消息**: 中文，格式 `[功能点] 实现/改进xxx`
- **提交验收**: 每次 `cargo build` + `cargo test` 通过
- **平台优先**: Linux (WSL2 Ubuntu 24.04)
- **跨平台**: Windows语法/命令工作，实现方式可不同

### 4. SSA架构维护

- **现状**: 优化器为NOP实现
- **保持**: 完整的Lexer → Parser → SSA IR → Executor流程
- **预留**: Phase 4实现SSA优化器
- **要求**: 所有功能都需在SSA IR中表示清晰
